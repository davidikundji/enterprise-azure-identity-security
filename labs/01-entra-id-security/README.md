# Lab 1 – Microsoft Entra ID Identity Management

## Overview

This lab demonstrates how to configure and manage identities using Microsoft Entra ID. Identity management is a core component of cloud security and ensures users have secure and appropriate access to resources.

---

## Objectives

- Create and manage users in Microsoft Entra ID  
- Assign roles and permissions  
- Understand identity lifecycle management  
- Monitor identity activity  

---

## Technologies Used

- Microsoft Azure  
- Microsoft Entra ID  

---

## Implementation Steps

### Step 1 – Access Microsoft Entra ID

1. Sign in to the Azure Portal  
2. Navigate to **Microsoft Entra ID**  
3. Open the **Users** section  

---

### Step 2 – Create Users

Create the following users:

- cloudadmin  
- securityuser  
- testemployee  

Each user is configured with:

- Username and domain  
- Temporary password  
- Assigned roles  

---

### Step 3 – Assign Roles

Assign roles based on responsibilities:

- Global Administrator (for admin account)  
- User Administrator  
- Standard user role  

This demonstrates **Role-Based Access Control (RBAC)**.

---

### Step 4 – Verify Identity Configuration

- Confirm users appear in Entra ID  
- Verify role assignments  
- Ensure login functionality  

---

## Security Concepts Demonstrated

### Identity Lifecycle Management
Managing users from creation to deletion.

### Role-Based Access Control (RBAC)
Assigning permissions based on roles.

### Authentication
Ensuring secure user access to systems.

---

## Lab Outcome

After completing this lab:

- Users are successfully created in Microsoft Entra ID  
- Roles and permissions are assigned  
- Identity management is configured for a cloud environment
